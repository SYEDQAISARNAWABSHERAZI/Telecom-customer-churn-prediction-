# Telco Customer Churn Prediction System

A complete **Machine Learning + Flask web application** for predicting whether a telecom customer is likely to **churn (leave the service)** based on customer demographics, subscription details, billing behavior, and service usage patterns.

This project was developed as a **Final Year Machine Learning Project** and demonstrates the **full ML lifecycle**:

* data preprocessing
* feature engineering
* model comparison
* model evaluation
* visualization
* Flask deployment
* business interpretation through feature importance

---

## Project Overview

Customer churn is a major business problem in the telecom industry because losing customers directly impacts **revenue, customer retention, and acquisition cost**.

This project predicts whether a customer is likely to churn using historical telecom customer data and provides a web-based interface where users can:

* enter customer information
* get churn prediction instantly
* view churn probability
* analyze dashboard charts
* inspect model feature importance
* review business risk interpretation and retention recommendations

## Machine Learning Workflow

### 1. Data Preprocessing

The dataset is cleaned and prepared by:

* removing unnecessary columns such as `customerID`
* converting `TotalCharges` to numeric format
* handling missing values
* preparing the target variable (`Churn`) for model training

### 2. Feature Engineering

Additional features were created to improve model performance and business interpretation:

* **tenure_group** → groups customers by retention duration
* **service_count** → total number of telecom services used by a customer

These engineered features help the model better understand **customer engagement patterns**.

### 3. Model Training & Comparison

The machine learning pipeline includes:

* numeric preprocessing
* categorical preprocessing
* feature encoding
* scaling
* model fitting
* multi-model comparison

The project compares:

* **Logistic Regression**
* **Random Forest Classifier**
* **XGBoost Classifier**

The **best model is automatically selected based on ROC-AUC**.

### 4. Model Evaluation

The model is evaluated using:

* Accuracy
* Precision
* Recall
* F1 Score
* ROC-AUC Score

### 5. Model Deployment

The trained model is integrated into a **Flask web app** for real-time churn prediction.

## Final Model Performance

These are the current results from training:

| Metric    |  Score |
| --------- | -----: |
| Accuracy  | 0.7332 |
| Precision | 0.4989 |
| Recall    | 0.7932 |
| F1 Score  | 0.6125 |
| ROC-AUC   | 0.8363 |

### Best Model Selection

> The final deployed model is selected automatically based on **ROC-AUC**.

### Why These Metrics Matter

For churn prediction, **accuracy alone is not enough**.

* **Precision** tells how many predicted churners were actually churners
* **Recall** tells how many real churners were correctly identified
* **F1 Score** balances precision and recall
* **ROC-AUC** measures overall discrimination ability

### Interpretation of Results

The model achieved a **strong ROC-AUC of 0.8363**, which indicates good class discrimination. However, recall is comparatively lower, meaning some actual churners are still missed.

This makes the system useful for churn analysis, while also showing clear room for future improvement.

## Installation & Setup

### 1. Clone the Repository

```bash
git clone https://github.com/fahadrazzaqcsventures/telco-churn-ml-flask-app.git
cd telco-churn-ml-flask-app
```

### 2. Create Virtual Environment

#### Linux / macOS

```bash
python3 -m venv venv
source venv/bin/activate
```

#### Windows

```bash
python -m venv venv
venv\Scripts\activate
```

### 3. Install Dependencies

```bash
pip install -r requirements.txt
```

### 4. Add Dataset

Place your dataset file here:

```bash
data/Telco-Customer-Churn.csv
```

### 5. Train the Models

```bash
python -m src.train
```

### 6. Generate Charts & Feature Importance

```bash
python -m src.plots
```

### 7. Run Flask App

```bash
python app.py
```

Then open:

```bash
http://127.0.0.1:5000
```
